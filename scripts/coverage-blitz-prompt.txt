## RULES — read these carefully

1. **All tests run against RESP2.** Use `GLOBAL.SERVERS.OPEN` (or `GLOBAL.CLUSTERS.OPEN` for cluster commands). This is the RESP2 server. Do NOT use any RESP3 server config.

2. **Do NOT duplicate existing tests.** If an existing `testWithClient`, `testAll`, or `testWith*` already exercises the command with data and asserts the response structure, that is sufficient. Do not add another test that does the same thing.

3. **Do NOT rename existing test descriptions.** Leave them exactly as they are.

4. **Do NOT rewrite or remove existing tests.** Only add new test cases alongside existing ones if coverage is missing.

5. **What counts as sufficient structural coverage:**
   - `assert.deepEqual(reply, { ... })` or `assert.deepStrictEqual(reply, { ... })` — **sufficient**, this pins the exact shape.
   - `assert.ok(Array.isArray(reply))` followed by assertions on element structure — **sufficient**.
   - `assert.equal(typeof reply, 'object')` or `assert.ok(reply !== null)` — **NOT sufficient**, this is too weak to catch Array→Map or other shape changes.
   - No integration test at all (only `transformArguments` unit tests) — **NOT sufficient**.

6. **If coverage is already sufficient, change NOTHING.** Go straight to the summary with STATUS: ALREADY_COVERED.
